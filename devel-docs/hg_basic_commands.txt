Здесь указаны несколько базовых команд системы контроля версий Mercurial (Hg), которые понадобятся при работе над проектом KEDR.

1. 
Создание копии основного (центрального, эталонного, pristine) репозитория, в котором идёт разработка очередной версии системы KEDR:
    hg clone https://itgdev/hgwebdir.cgi/kedr/kedr-0.x/current/ kedr-current

В результате будет создан каталог 'kedr-current', содержащий копию соотв. репозитория. 

При жедании можно создать и другие локальные копии и из этой свежесозданной копии тоже:
    hg clone kedr-current kedr-current-other

2.
Получение изменений из репозитория, откуда была сделана данная копия (pull):
    hg pull

3.
Проверка статуса файлов:
    hg status

4.
Добавление файлов:
    hg add

5.
Получение списка изменений:
    hg log

6.
"Фиксация" изменений в локальном репозитории:
    hg commit

7. 
Внесение результатов работы на локальной машине в центральный репозиторий.

Полезно перед этим просмотреть список изменений по сравнению с центр. репозиторием с помощью
    hg outgoing

Короткий вариант (если в центральном репозитории никаких изменений не было с момента hg pull):
    hg push

Длинный (но часто необходимый вариант):
    hg pull -u # загрузить последние изменения из центр. репозитория
    hg merge   # нужно, если в центр. репозитории были другие изменения
    hg commit  # commit для результатов merge
    hg push    # загрузить результаты в центр. репозиторий

=======================================================================

Подробнее о командах Mercurial см. в документации по Mercurial ("Mercurial: The Definitive Guide"). Документация эта входит, в частности в поставку TortoiseHg (http://tortoisehg.bitbucket.org/), а также доступна на офиц. сайте Mercurial: http://mercurial.selenic.com/

Также может пригодиться встроенная справка Mercurial:
    hg help <имя команды>
